Set Box-Volume pair

The first step is to find the box volume pair for the local machine.

> D GETENV^%ZOSV W Y

which will print out a message with four parts separated by ^ that could look something like:

VISTA^VISTA^palaven^VISTA:CACHE

The four pieces of the string are:

UCI^VOL^NODE^VOLUME:BOX

The fourth piece, the VOLUME:BOX pair, is referred to as the "BOX VOLUME pair". The first component of the Box Volume pair is the Volume Set, which is used to determine where the VistA system will be able to find the routines. The second component Box, which references the system that the instance is on. In a Caché system, it would be the name of the Caché instance while on GT.M, it should reference the hostname of the machine.

The Volume Set result needs to be altered in the VOLUME SET file, and we will reuse some setup by writing over the name of the first entry that is already in the VistA system. The first entry, the entry with an IEN of 1, can be selected by entering `1.

Then we rename the first Box-Volume pair in the TaskMan Site Parameters file to match what was found above.

For this demonstration instance, the Volume Set will be VISTA which is the Caché namespace that holds the files. On GT.M instances, the default value of PLA can be maintained.

> S DUZ=.5 D Q^DI VA FileMan 22.0 Select OPTION: 1 ENTER OR EDIT FILE ENTRIES INPUT TO WHAT FILE: DEVICE// 14.5 VOLUME SET (1 entry) EDIT WHICH FIELD: ALL// VOLUME SET THEN EDIT FIELD: TASKMAN FILES UCI THEN EDIT FIELD: <enter> Select VOLUME SET: `1 PLA VOLUME SET: PLA// VISTA TASKMAN FILES UCI: PLA// VISTA Select VOLUME SET: <enter> INPUT TO WHAT FILE: TASKMAN SITE PARAMETERS// 14.7 TASKMAN SITE PARAMETERS (1 entry) EDIT WHICH FIELD: ALL// <enter> Select TASKMAN SITE PARAMETERS BOX-VOLUME PAIR: \`1 PLA:PLAISCSVR BOX-VOLUME PAIR: PLA:PLAISCSVR// VISTA:CACHE RESERVED: ^ Select TASKMAN SITE PARAMETERS BOX-VOLUME PAIR: <enter> Select OPTION: <enter>

For DEFAULT TIMED-READ (SECONDS): if you change it to 3600 you will be allow an hour before being automatically signed off. It makes it easier to work when you are learning and setting things up.

Press <PF1> refers to notations for use of Vista on Terminals. For example, the original VT-320 keyboard had additional character sets and keys which include Find, Select, Insert, Remove, Previous Screen, Next Screen, an arrow cluster and F1 to F20. With Caché, the Keys are "mapped", which means when you push a given key it acts as the key would in a terminal. For instance, F1, F2, F3 and F4 are equivalent to the PF1, PF2, PF3 and PF4 keys on the terminal keyboard and Page Up and Page Down on the computer keyboard correspond to Previous Screen and Next Screen. A listing of other mappings can be found at the Caché Cube Terminal window under Help and search Keyboard Mappings. Also there is lots of information about terminal if you are interested at www.VT100.net.
